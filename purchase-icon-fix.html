🛒 购买页面专区图标显示修复

✅ 问题已修复！

问题：购买卡密页面的专区图标只显示emoji，不显示自定义上传的图片。

解决：修改模板逻辑，支持同时显示emoji图标和自定义图片，与管理页面保持一致。

🔍 问题分析

根本原因

购买页面(`CardPurchase.vue`)的专区图标显示逻辑过于简单，只支持emoji图标：

// 原问题代码（第94行） <div class="icon-wrapper">{{ zone.icon }}</div> // 这种方式只能显示emoji图标，无法显示自定义图片

数据流分析

  1. 管理员在后台上传自定义图标 → 保存到 zone.customIcon 字段
  2. 购买页面从 localStorage 加载专区数据 → 数据是完整的
  3. 但是模板只使用 zone.icon 显示 → 忽略了 zone.customIcon

🔧 修复详情

❌ 修复前

<div class="category-icon"> <div class="icon-wrapper"> {{ zone.icon }} </div> </div> /* 只支持emoji显示 */

✅ 修复后

<div class="category-icon"> <div class="icon-wrapper"> <!-- 智能图标显示 --> <span v-if="!zone.customIcon || zone.customIcon.trim() === ''"> {{ zone.icon || '🎯' }} </span> <img v-else :src="zone.customIcon" :alt="zone.name" class="custom-zone-icon" @error="handleIconError(zone)" /> </div> </div> /* 支持emoji和自定义图片 */

修复步骤详解

1 修改模板显示逻辑

使用条件渲染，优先显示自定义图标，fallback到emoji图标：

<!-- 显示自定义图标或emoji图标 --> <span v-if="!zone.customIcon || zone.customIcon.trim() === ''">{{ zone.icon || '🎯' }}</span> <img v-else :src="zone.customIcon" :alt="zone.name" class="custom-zone-icon" @error="handleIconError(zone)" />
2 添加自定义图标样式
.icon-wrapper { width: 72px; height: 72px; background: linear-gradient(135deg, #f8fafc 0%, #e2e8f0 100%); border: 1px solid rgba(15, 23, 42, 0.1); border-radius: 16px; // 自定义图标样式 .custom-zone-icon { width: 100%; height: 100%; object-fit: cover; // 保持比例，填充容器 border-radius: 16px; // 与容器圆角一致 } }
3 添加错误处理机制
// 图片加载错误处理 const handleIconError = (zone) => { console.warn(`专区 ${zone.name} 的自定义图标加载失败，恢复为emoji图标`) zone.customIcon = '' // 清空失效的图标 if (!zone.icon) { zone.icon = '🎯' // 设置默认emoji } }
4 增强数据加载调试
const loadData = () => { // ...加载逻辑... console.log('🛱 购买页面加载专区数据:', zones.value.length, '个活跃专区') // 检查每个专区的图标数据 zones.value.forEach((zone, index) => { console.log(`专区 ${index + 1}:`, { id: zone.id, name: zone.name, icon: zone.icon || '无', hasCustomIcon: !!zone.customIcon, customIconLength: zone.customIcon ? zone.customIcon.length : 0 }) }) }

  3. 访问购买卡密页面
  4. 查看控制台日志，确认专区数据加载正确：

    • 有自定义图标的专区 → 显示上传的图片

    • 图标加载失败 → 自动回退到emoji或默认图标

🔧 技术细节

条件渲染逻辑

// 判断是否有有效的自定义图标 v-if="!zone.customIcon || zone.customIcon.trim() === ''" // 条件分解： // 1. !zone.customIcon → customIcon字段不存在或为null/undefined // 2. zone.customIcon.trim() === '' → customIcon存在但为空字符串或只有空格

图片样式适配

.custom-zone-icon { width: 100%; // 填满容器宽度 height: 100%; // 填满容器高度 object-fit: cover; // 保持宽高比，裁剪适应容器 border-radius: 16px; // 圆角与容器一致 }

错误处理机制

@error="handleIconError(zone)" // 当图片加载失败时： // 1. 清空失效的customIcon // 2. 确保有fallback的emoji图标 // 3. 输出警告日志便于调试
